# update-board — 对话进度 → 项目内固定路径的自包含看板 HTML

把**当前对话/项目**的进度,生成一个**自包含 HTML**(数据+样式全内联),
写到**当前项目目录下的固定相对路径**:`.board/board.html`。

## 核心约定(关键)
- **一个项目 / 一个会话空间 → 一个 `.board/board.html`**。不同项目在各自目录下,互不干扰。
- 统一的不是"某个全局文件",而是**相对路径规则**:每个项目目录下,看板永远位于 `.board/board.html`。
- 用户的看板已知道每个项目的根目录,因此按 `<项目根>/.board/board.html` 加载即可,路径规则统一:
  ```html
  <iframe src="<项目根>/.board/board.html" style="width:100%;height:1000px;border:0"></iframe>
  ```
- 用户**不需要知道项目名/slug**;路径由"项目根 + 固定相对路径"决定。

## 执行步骤
1. **提炼进度**:通读当前对话,归纳 —— 项目标题/副标题、当前阶段、关键指标(KPI)、各任务/模块及状态(done/wip/todo)与明细、下一步。**只写有实据的,`done` 必须真完成**;日期用当前绝对日期。
2. **生成自包含 HTML**,写到当前工作目录的 `.board/board.html`:
   - **自包含**:`<style>` 内联、数据直接写进 HTML,**不要 fetch 外部文件**(保证 file:// 和任意内嵌都能直接显示)。
   - 统一深色视觉:顶部标题 + 阶段标签 + KPI 卡 + 任务卡片(图标/状态徽章/进度条/明细列表)+ 下一步。
   - 若文件已存在:基于其内容更新(保留未变项、改有变项),不丢信息。
3. **校验并汇报**:确认已写 `.board/board.html`;给出整体进度,以及内嵌用的 iframe 片段。

## 原则
- **一项目一文件 + 固定相对路径 + 自包含**:用户看板按 `<项目根>/.board/board.html` 加载,规则统一、各项目隔离、无需知道项目名、无 CORS/服务器依赖。
- **诚实**:进度如实反映对话。
- **观感统一**:所有项目沿用同一套样式,换项目只是内容不同。
